## ① 기능 목적

### 이 기능이 해결하는 사용자 문제

- “나는 어떤 책을 좋아하는지 잘 모르겠다”
- “추천을 받아도 나랑 맞는지 확신이 안 든다”
- “이 서비스에서 어디부터 시작해야 할지 모르겠다”

👉 이 기능은

**사용자의 독서 성향을 가볍게 정의해 주고, 나와 비슷한 독서 맥락을 가진 ‘대표 인물’을 제시함으로써 서비스 진입 장벽을 낮춘다.**

---

## ② 필요한 핵심 데이터

### A. 테스트 설계 데이터

| 필드명 | 설명 |
| --- | --- |
| question_id | 질문 고유 ID |
| question_text | 질문 문항 |
| answer_id | 선택지 ID |
| answer_text | 선택지 문구 |
| answer_weight | 선택지별 가중치 |

---

### B. 독서 성향 결과 데이터

| 필드명 | 설명 |
| --- | --- |
| dna_type_id | 독서 DNA 유형 ID |
| dna_type_name | 독서 DNA 이름 (ex. 현실 설계형, 사유 확장형 등) |
| dna_description | 유형 설명 문구 |
| dna_keywords | 성향 키워드 |
| matched_person_id | 대표 매칭 인물 ID |
| matched_person_name | 매칭 인물 이름 |

---

### C. 데이터 성격 구분

- **유저 생성 데이터**
    - 테스트 응답
    - 도출된 dna_type_id
- **외부/운영 데이터**
    - 질문/선택지
    - DNA 유형 정의
    - DNA–대표 인물 매핑 정보

---

## ③ 데이터 수집 방식

- **수동 입력 (고정)**
    - 질문 문항 작성
    - 선택지 설계
    - DNA 유형 정의
    - DNA ↔ 인물 매칭 설정

---

## ④ 세부 기능 목록

### A. 화면에 보이는 것

### 1️⃣ 테스트 시작 화면

- “몇 가지 질문으로 나와 비슷한 독서 스타일을 알아보세요”
- 질문 수 표시 (예: 6문항, 1분 소요)
- 시작 버튼

---

### 2️⃣ 질문 응답 화면

- 질문 1개씩 순차 노출
- 선택지는 2~4개
- 진행률 표시

예시 질문:

- “책을 읽다가 이런 순간에 가장 멈추게 된다면?”
- “요즘 책을 읽고 싶은 이유에 가장 가까운 것은?”

---

### 3️⃣ 결과 화면

**구성**

- 독서 DNA 유형 이름
- 한 줄 설명
- 키워드 2~3개
- **매칭된 대표 인물**
    - 인물 이미지
    - 이름
    - “이 사람과 비슷한 독서 맥락을 가졌어요”
- CTA
    - “이 인물의 서재 보러 가기”
    - “이 유형에게 추천된 책 보기”

---

### B. 내부 로직

### 1️⃣ 응답 집계 로직

- 각 선택지는 특정 성향 축에 가중치 부여
- 응답 누적 → 가장 높은 축이 dna_type 결정

---

### 2️⃣ DNA 유형 매칭 로직

- dna_type_id ↔ 대표 인물 1명 매칭
- 추후: 서브 인물 여러 명 매칭 가능

---

### 3️⃣ 다른 기능과의 연결

- 인물 중심 탐색 → 테스트 결과에서 바로 해당 인물 서재로 이동
- 스낵 알림 → DNA 유형 기반 화자 선정
- 개인 서재 → 사용자 기본 독서 성향 저장
