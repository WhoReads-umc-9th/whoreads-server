## ① 기능 목적

### 이 기능이 해결하는 사용자 문제

- “무슨 책을 읽어야 할지 모르겠다”
- “책 설명보다, 내가 좋아하는 사람이 왜 읽었는지가 궁금하다”
- “서점이나 베스트셀러 추천은 나랑 잘 안 맞는다”

👉 이 기능은

**책을 기준으로 고민하게 만드는 대신, ‘사람’을 기준으로 자연스럽게 독서에 진입**하게 한다.

즉,

> 책 고르기를 *롤모델의 생각을 들여다보는 경험*으로 전환한다.
>

---

## ② 필요한 핵심 데이터

### A. 데이터 필드 정의

### 1) 인물 기본 정보

| 필드명 | 설명 |
| --- | --- |
| person_id | 인물 고유 ID |
| person_name | 인물 이름 |
| person_image_url | 인물 대표 이미지 URL |
| person_short_bio | 한 줄 소개 (직업·정체성 중심) |
| person_tags | 인물 키워드 (기업가, 작가, 아티스트 등) |

---

### 2) 인물-책 연결 데이터

※ **증거 기반 인용 & 딥링크(1번 기능)의 데이터 재사용**

| 필드명 | 설명 |
| --- | --- |
| book_id | 책 고유 ID |
| book_title | 책 제목 |
| quote_id | 대표 인용 ID |
| quote_original | 대표 인용 원문 |
| confidence_score | 대표 인용 신뢰도 점수 |
| context_score | 대표 인용 맥락 점수 |
| source_url | 대표 인용 출처 링크 |
| source_type | 출처 유형 |

📌 **중요 정책**

- 동일 `book_id`는 **인물 서재에서 1회만 노출**
- 동일 책에 여러 인용이 있을 경우: 내부 점수 기준으로 **대표 인용 1개 선정** *(아래에 자세히 설명)*

---

### B. 데이터 성격 구분

- **외부 데이터**
    - 인물 프로필 정보
    - 인물 이미지
    - 인물 발언 기반 인용 데이터
- **유저 생성 데이터**
    - 없음 (탐색 기능은 큐레이션 영역)

---

## ③ 데이터 수집 방식

- **수동 입력 (고정)**
    - 인물 선정 및 프로필 작성
    - 인물 이미지 파일 업로드
    - 인용 데이터 연결

📌 전략

- 초기에는 **소수 인물 중심의 고밀도 서재**
- “많이”보다 **“신뢰 가능한 사람”** 우선

---

## ④ 세부 기능 목록

### A. 화면에 보이는 것

### 1️⃣ 인물 탐색 화면 (리스트)

- 인물 카드
    - 인물 이미지
    - 인물 이름
    - 태그(분야)
    - 한 줄 소개

📌 사용자가 **“이 사람 뭐 하는 사람이지?”를 즉시 파악**할 수 있어야 함

---

### 2️⃣ 인물 상세 화면 = 가상 서재

**구성**

- 상단 영역
    - 인물 이미지
    - 인물 이름
    - 한 줄 소개
- 서재 영역
    - 이 인물이 언급한 책 리스트
    - 각 책은 **1회만 노출**
    - 책 카드 구성:
        - 책 제목
        - 대표 인용 원문 (말줄임표 적용)
        - “출처 확인” 버튼

---

### B. 내부 로직

### 1️⃣ 대표 인용 선정 로직 (중요)

동일 책에 인용이 여러 개 있을 경우, 아래 순서로 1개 선택:

1. confidence_score 높은 인용
2. context_score 높은 인용
3. 출처 신뢰도 (영상 > 인터뷰 > SNS > 기사)
4. 최신 출처

---

### 2️⃣ 책 정렬 로직

- 기본 정렬:
    - 대표 인용의 confidence_score 내림차순
    - 대표 인용의 context_score 내림차순
- 신뢰도와 맥락이 명확한 책이 상단 노출

---

### 3️⃣ 다른 기능과의 연결

- 주제별 큐레이션 → 여러 인물의 가상 서재 데이터를 교차 활용
- 스낵 알림 → 대표 인용 원문 일부 사용
- 개인 서재(추후) → 사용자가 팔로우한 인물 기반 추천
