## ① 기능 목적

### 이 기능이 해결하는 사용자 문제

- “이 사람이 정말 이 책을 읽은 게 맞는지 신뢰하기 어렵다”
- “왜 이 책을 읽었는지, 어떤 상황이었는지 알 수 없다”
- “나한테 안 맞는 책을 고를까 봐 독서를 시작하지 못한다”

👉 이 기능은

**유명인의 실제 발언 원문과 명확한 출처를 제공해 독서 추천의 신뢰도를 확보하고, 사용자의 ‘선택 실패에 대한 두려움’을 줄인다.**

---

## ② 필요한 핵심 데이터

### A. 데이터 필드 정의

### 1) 인용 식별 및 연결 필드

| 필드명 | 설명 |
| --- | --- |
| quote_id | 인용 데이터 고유 식별자 |
| person_id | 인물 고유 ID (동명이인 구분) |
| person_name | 서비스 노출용 인물 이름 |
| person_image_url | 인물 이미지 파일을 스토리지에 업로드한 뒤 사용하는 URL |
| book_id | 책 고유 ID |
| book_title | 책 제목 |

---

### 2) 인용 원문 데이터

| 필드명 | 설명 |
| --- | --- |
| quote_original | 실제 발언 원문 (가공하지 않은 텍스트) |
| quote_language | 원문 언어 (ko / en 등) |
- 인용이 길 경우 UI에서 말줄임표 + 펼쳐보기로 처리

---

### 3) 독서 맥락 데이터 (내부 구조화용)

| 필드명 | 설명 |
| --- | --- |
| context_how | 이 책을 어떻게 접하게 되었는지 (계기) |
| context_when | 어떤 시기·상황에서 읽었는지 |
| context_why | 그 시점에 왜 이 책이 필요했는지 |
| context_help | 이 책이 어떤 도움이나 변화를 주었는지 |
- 해당 필드들은 **화면에 직접 노출되지 않음**
- 정렬, 필터, 라벨, 알림 문구 생성 등 **서비스 내부 로직**에 사용

---

### 4) 출처 및 증거 필드

| 필드명 | 설명 |
| --- | --- |
| source_url | 인용의 원문 링크 |
| source_type | 출처 유형 (sns / interview / video / article 등) |
| source_timestamp | 영상일 경우 발언 시작 시점 |
| source_title | 인터뷰·영상 제목 |
| source_date | 발언 날짜 (확인 가능할 경우) |

---

### 5) 신뢰도 판단 메타 필드

**① 사실 기록용 (boolean)**

| 필드명 | 설명 |
| --- | --- |
| is_first_person | 인물 본인의 1인칭 발언 여부 |
| is_direct_quote | 직접 인용 여부 |
| is_ad | 광고·협찬 여부 |

**② 점수 기록용 (number)**

| 필드명 | 설명 |
| --- | --- |
| confidence_score | 인용 신뢰도 종합 점수 (0–5) |
| context_score | 독서 맥락 명확도 점수 (0–5) |
- boolean은 **근거 기록용**
- score는 **노출·정렬 판단용**
- 점수 기준은 운영 가이드에 따라 일관되게 산정

---

### B. 데이터 성격 구분

- **외부 데이터**
    - 인물 발언 원문
    - 출처 링크
    - 인물 이미지 파일(jpg/png → 스토리지 업로드 후 URL 사용)
- **유저 생성 데이터**
    - 없음

      → 신뢰 확보를 위해 인용 데이터는 운영자 관리 영역으로 유지


---

## ③ 데이터 수집 방식

- **수동 입력 (고정)**
    - 운영자가 직접 링크 선별
    - 인용 원문 발췌
    - 독서 맥락(context_*) 필드 정리
    - 신뢰도 메타 필드 입력
    - 인물 이미지 파일 수집 및 업로드
- 링크 타입 제한 없음

  (SNS, 인터뷰, 영상, 기사 모두 가능)

- 광고·추천사 성격 콘텐츠는 낮은 우선순위로 관리

---

## ④ 세부 기능 목록

### A. 화면에 보이는 것

1️⃣ **인용 카드**

- 인물 이미지
- 인물 이름
- 책 제목
- 인용 원문 (말줄임표 적용)
- “출처 확인” 버튼

2️⃣ **출처 딥링크**

- 클릭 시 원문 페이지 이동
- 영상일 경우 해당 발언 시점으로 이동

---

### B. 내부 로직

1️⃣ **인용 정렬 로직**

- is_first_person = true 우선
- is_ad = false 우선
- confidence_score → context_score 순 정렬

2️⃣ **노출 분기**

- 신뢰도·맥락 점수 높은 인용: 메인 추천 영역
- 신뢰는 있으나 맥락이 약한 인용: 참고용 영역

3️⃣ **다른 기능과의 연결**

- 인물 중심 탐색(가상 서재): 인물별 인용 묶음
- 주제별 큐레이션: 인용 기반 책 정렬
- 스낵 알림: 인용 원문 일부 활용

---

## ⑤ 신뢰도 점수 기준 (운영·개발 공통 규칙)

### 1️⃣ confidence_score (신뢰도 점수, 0–5점)

> 이 인용이 사실로서 얼마나 믿을 수 있는가
>

| 기준 | 조건 | 점수 |
| --- | --- | --- |
| 본인 직접 발언 | is_first_person = true | +2 |
| 직접 인용 | is_direct_quote = true | +1 |
| 광고 아님 | is_ad = false | +1 |
| 출처 명확 | 원문 링크·발언 위치 확인 가능 | +1 |

📌 해석

- 4–5점: 신뢰도 높음
- 2–3점: 참고 가능
- 0–1점: 노출 비권장

---

### 2️⃣ context_score (맥락 명확도 점수, 0–5점)

> 이 인용이 ‘왜 이 책을 읽었는지’를 얼마나 설득력 있게 설명하는가
>

| 맥락 요소 | 기준 | 점수 |
| --- | --- | --- |
| context_how | 읽게 된 계기 명확 | +1 |
| context_when | 시기·상황 명확 | +1 |
| context_why | 필요했던 이유 명확 | +2 |
| context_help | 도움·변화 명확 | +1 |

📌 why에 2점을 주는 이유

→ 추천과 경험을 가르는 핵심 정보이기 때문

---

### 3️⃣ 점수 활용 원칙

- 점수는 **사용자에게 직접 노출되지 않음**
- 인용 정렬, 메인/참고 노출 분기, 알림 사용 여부 판단에만 사용
- 운영자가 바뀌어도 동일한 기준으로 판단 가능해야 함
